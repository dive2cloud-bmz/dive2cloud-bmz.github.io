alibaba cloud sls query language learning

sls

背景

在阿里云的这段日子，各种日志都是交给 sls 管理的，量很大，但是查询检索以及采集都很平滑（不愧为轩哥嘴里的明星项目）。
不会查询语法只会无脑写点单词去检索日志，感觉笨笨的，简单参照官网学习一下

基础

格式：查询语句|分析语句
查询语句可以单独用，但分析语句必须和查询语句结合

查询语句

可以是关键字，数值，数值范围，空格，星号(*)等，如果是空格和星号，则表示无过滤条件

分析语句

暂时不看，有需要自己看官网吧

https://help.aliyun.com/zh/sls/query-syntax/?spm=5176.2020520112.console-base_help.dexternal.34483efd7f7dDh

一个示例

  • | SELECT status, count(*) AS PV GROUP BY status

怎么写查询语句

确定查询方式

先要确定 logstore 索引类型是什么，分为全文索引和字段索引

  • 如果只创建了全文，则只能使用全文查询
  • 如果创建过字段索引，可以用一些字段查询语法

全文查询语法

格式： keywords1 [and | or | not] keywords2 …
示例：假如我要查询的关键字是 GET 和 POST

GET or POST

假如我要查询 Jo 开头的日志

Jo?

字段查询语法

可以类型化运算（数值比较、正则），需要字段建立过索引

假如 indexname1 是要查询的字段名，当字段名、表名等名词中有特殊字符（空格、中文）、关键字（or and等），需要用 双引号 包裹住。
字段索引涉及 long double 类型，可以用 > >= < <= = in 来比较

格式：indexname1 [ : | > | >= | < | <= | = | in ] keyword1 [[ and | or | not ] indexname2 …]
示例：假如我要查询 request_time_msec 大于 50 的日志，且 request_time_msec 已经建立过 double 类型的字段索引

request_time_mse>50

示例2：查询 request_method 为 GET 且request_time_msec 大于 50 相关的日志

request_method: GET and request_time_msec>50

确定字段类型

  1. text
    字符串类型， sls 默认将整条日志（除了 time 之外的字段）都设置成 text 类型
    可用运算符： and or not () : “” \ * ?

  2. long double
    设置索引时要指定字段，可以用 > < 哪些
    值得注意的是，假如有一个字段 owner_id 类型不是 double long，我还执行查询语句 owner_id>100 ，则会同时返回包含 owner_id > 100 这三个词的日志
    如果是一个字段是 text 改成的 double/long，则只支持等号查询，想要支持大于小于，必须重建索引

    in 必须小写

运算优先级由高到低

: > “” > () > and,not > or

运算符示例

  • 冒号：request_method:GET

  • and: request_method:GET and status:200

  • or: request_method:GET or status:200

  • not: request_method:GET not status:200

  • “”（双引号）: “tag:client_ip“:192.0.2.1

    因为 tag 是保留字，类似的保留字还有 sort, asc, desc, group by, avg, sum, min, max, limit

  • (转义，比如转义双引号 ""): instance_id:nginx"01"

  • 星号(*): 匹配0，1，多个字符

  • ?: 匹配1个字符

  • in: 专门查数值 request_time in [100 200] 或 request_time in (100 200]，大括号是开区间，中括号是闭区间
